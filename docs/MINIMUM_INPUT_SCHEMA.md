# IPK 그룹웨어 최소 입력 스키마

> **Agent**: /문서분석
> **분석일**: 2025-12-29
> **목적**: 사용자가 최소한의 입력만으로 문서를 자동 작성할 수 있도록 필드 분류

---

## 분석 개요

### 분석 대상
- 4개 문서 유형: Leave, Expense, Working, Travel
- 승인 완료 문서: 25개 샘플
- 빈 템플릿 구조 분석

### 분류 기준
| 분류 | 설명 | 처리 방법 |
|------|------|----------|
| 🔴 필수 입력 | 사용자 의도 필요 | CLI 인자로 받음 |
| 🟢 자동 추론 | 시스템/세션 정보 | 자동 채움 |
| 🔵 팝업 기반 | UI 상호작용 필요 | 자동화된 팝업 처리 |
| 🟡 이력 기반 | 과거 패턴에서 추론 | 최빈값 또는 최근값 사용 |

---

## 1. Leave (휴가신청서)

### 템플릿 구조
- Input 필드: 35개
- Select 필드: 7개
- 팝업 버튼: 3개

### 🔴 사용자 필수 입력 (5개)

| 필드명 | 설명 | 추론 불가 이유 |
|--------|------|----------------|
| `leave_kind` | 휴가 유형 | 의도 필요 (연차/보상/병가/경조사 등) |
| `begin_date` | 시작일 | 날짜 의도 필요 |
| `end_date` | 종료일 | 날짜 의도 필요 |
| `purpose` | 휴가 사유 | 개인 정보 |
| `destination` | 목적지 | 개인 정보 |

### 🟢 자동 추론 가능 (6개)

| 필드명 | 추론 방법 | 근거 |
|--------|----------|------|
| `subject` | 템플릿 생성 | "{휴가유형}, {시작일}~{종료일}, {목적지}, {이름}" |
| `drafter` | 세션 정보 | 로그인 사용자 자동 |
| `department` | 세션 정보 | 사용자 프로필 |
| `document_no` | 시스템 생성 | readonly 필드 |
| `close_date` | 시스템 자동 | readonly 필드 |
| `retention` | 기본값 | "1 year" 기본 선택 |

### 🔵 팝업 선택 기반 (1개)

| 필드명 | 팝업 기능 | 자동화 방법 |
|--------|----------|-------------|
| `substitute` | 대리자 선택 | 이름 검색 → 라디오 클릭 → OK |

### 🟡 이력 기반 추론 (4개)

| 필드명 | 추론 방법 | 신뢰도 |
|--------|----------|--------|
| `substitute` | 최근 5회 최빈값 | 90% |
| `destination` | 유형별 최빈값 | 80% |
| `purpose` | 유형별 패턴 | 70% |
| `emergency_contact` | 고정값 저장 | 95% |

### 첨부파일 요구사항
| 휴가 유형 | 코드 | 첨부 필수 |
|----------|------|-----------|
| 병가 | 02 | **필수** |
| 공가 | 03 | **필수** |
| 특별휴가 (경조사) | 05 | **필수** |
| 가족돌봄휴가 | 07 | **필수** |
| 기타 공가 | 13 | **필수** |
| 난임치료휴가 | 15 | **필수** |
| 연차/보상휴가 | 01,04 | 선택 |

### 최소 CLI 명령
```bash
# 연차 (첨부 불필요)
ipk leave --type annual --date 2025-01-05 --reason "개인 사유"

# 병가 (첨부 필수)
ipk leave --type sick --date 2025-01-05 --reason "병원 방문" \
    --attachment diagnosis.pdf
```

## 2. Expense (경비청구서)

### 템플릿 구조
- Input 필드: 759개 (행 반복)
- Select 필드: 8개
- 팝업 버튼: 2개

### 🔴 사용자 필수 입력 (5개)

| 필드명 | 설명 | 추론 불가 이유 |
|--------|------|----------------|
| `budget_type` | 예산 유형 | General vs R&D 선택 필요 |
| `budget_code` | 예산 코드 | 프로젝트별 다름 |
| `item_name` | 항목명 | 청구 내역 기술 필요 |
| `item_amount` | 금액 | 영수증 금액 |
| `attachment` | 영수증 | **항상 필수** |

### 🟢 자동 추론 가능 (5개)

| 필드명 | 추론 방법 | 근거 |
|--------|----------|------|
| `subject` | 템플릿 생성 | 항목명 기반 |
| `drafter` | 세션 정보 | 로그인 사용자 |
| `document_no` | 시스템 생성 | readonly |
| `retention` | 기본값 | "1 year" |
| `pay_kind` | 기본값 | "Personal Reimbursement" |

### 첨부파일 요구사항
```javascript
// JavaScript 검증 코드 (실제 추출)
if(attach_file_cnt == 0){
    alert('Please attach the file.');
    return false;
}
```
**결론**: 경비청구서는 **항상** 최소 1개 첨부 필수

### 최소 CLI 명령
```bash
# 항상 첨부 필수!
ipk expense --budget R&D --code FS2214-0001 \
    --item "회의비" --amount 50000 \
    --attachment receipt.pdf
```

- 팝업 버튼: 2개

### 🔴 사용자 필수 입력 (4개)

| 필드명 | 설명 | 추론 불가 이유 |
|--------|------|----------------|
| `work_date` | 근무일 | 날짜 의도 필요 |

| `budget_code` | 예산 코드 | 프로젝트별 다름 |
| `sub_subject` | 업무 내용 | 업무 기술 필요 |

### 🟢 자동 추론 가능 (4개)

| 필드명 | 추론 방법 | 근거 |

## 4. Travel (출장결과보고서)

### 템플릿 구조
- Input 필드: 36개
- Select 필드: 4개
- 팝업 버튼: 2개

### 🔴 사용자 필수 입력 (2개)

| 필드명 | 설명 | 추론 불가 이유 |
|--------|------|----------------|
| `pdoc_id` | 출장신청서 연결 | 이전 문서 선택 필요 |
| `summary` | 결과 요약 | 출장 결과 기술 필요 |

### 🟢 자동 추론 가능 (5개)

| 필드명 | 추론 방법 | 근거 |
|--------|----------|------|
| `report_date` | 연결 문서 | 출장신청서 종료일 |
| `destination` | 연결 문서 | 출장신청서 방문처 |
| `purpose` | 연결 문서 | 출장신청서 목적 |
| `subject` | 템플릿 생성 | 출장신청서 제목 기반 |
| `drafter` | 세션 정보 | 로그인 사용자 |

### 🔵 팝업 선택 기반 (1개)

| 필드명 | 팝업 기능 | 자동화 방법 |
|--------|----------|-------------|
| `pdoc_id` | 출장신청서 검색 | 문서 검색 팝업 → 선택 |

### 첨부파일 요구사항
- **권장**: 출장 관련 증빙 (영수증, 사진 등)
- 필수 아님 (JavaScript 검증 없음)

### 최소 CLI 명령
```bash
# 출장신청서 doc_id 연결로 대부분 자동 추론
ipk travel --ref-doc 286841 --summary "미팅 완료, 협력 논의"
```
