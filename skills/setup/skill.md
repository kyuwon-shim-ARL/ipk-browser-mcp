# IPK Browser MCP Setup

Configure IPK Groupware credentials for the ipk-browser-mcp plugin.

## Trigger

When user says: "ipk setup", "setup ipk", "configure ipk", "ipk 설정"

## Instructions

Guide the user through credential configuration. Credentials are stored at `~/.config/ipk-browser-mcp/.env` (never in the repo).

### Steps

1. Ask the user for their IPK Groupware credentials:
   - `IPK_USERNAME` (required) — groupware login ID
   - `IPK_PASSWORD` (required) — groupware password
   - `IPK_BASE_URL` (optional, default: `https://gw.ip-korea.org`)

2. Create the config directory and write the `.env` file:

```bash
mkdir -p ~/.config/ipk-browser-mcp
cat > ~/.config/ipk-browser-mcp/.env << 'ENVEOF'
# IPK Groupware Credentials
# Generated by /ipk-browser-mcp:setup
IPK_USERNAME=<user-provided>
IPK_PASSWORD=<user-provided>
IPK_BASE_URL=<user-provided-or-default>
ENVEOF
chmod 600 ~/.config/ipk-browser-mcp/.env
```

3. Verify the file was created:
```bash
ls -la ~/.config/ipk-browser-mcp/.env
```

4. Inform the user:
   - Credentials are stored securely at `~/.config/ipk-browser-mcp/.env` (mode 600)
   - The MCP server reads from this file automatically
   - To update credentials, run `/ipk-browser-mcp:setup` again
   - Restart Claude Code session to pick up new credentials

### Security Notes

- NEVER echo or display the password back to the user
- NEVER commit credentials to git
- File permissions are set to 600 (owner read/write only)
- The `.env` path is outside any git repository
